# Reinforcement Learning with LLM-Based Reward Feedback in a CoinGrid Environment

> **Master's Thesis Project** — Mounika Buddharaju

A curriculum-trained PPO agent that learns to follow **natural-language instructions** in a custom grid-world environment, with rewards generated by a locally-running Large Language Model (Gemma 3 via Ollama).

---

## Overview

Traditional RL reward functions are hand-crafted and brittle — they require engineers to manually define every success condition.  This project investigates whether a **pre-trained LLM can serve as a general-purpose reward model**, evaluating agent behaviour from natural language alone.

The agent navigates a 5×5 grid, collecting coloured coins (red, green, blue) according to instructions like:

> *"Pick up 2 red coins and 1 blue coin"*
> *"Ensure you fetch 1 green coin"*

A **10-stage curriculum** progressively transfers from dense, rule-based reward shaping to sparse, fully LLM-generated rewards — enabling the agent to generalise to instructions it has never seen before.

## Curriculum

The agent is trained through 10 progressive stages.  Early stages use a fully deterministic environment and hand-crafted rewards; later stages introduce randomness and replace Python rewards with LLM feedback.

| Stage | Layout | Instruction | Reward |
|---|---|---|---|
| 1 | Fixed | Fixed | Stepwise |
| 1.1 | Fixed | Templated · 1 colour | Stepwise |
| 1.2 | Fixed | Templated · 2 colours | Stepwise |
| 1.3 | Fixed | Templated · 3 colours | Stepwise |
| 2 | Fixed | Random | Stepwise |
| 2.1 | 3-fixed + random | Fixed | Stepwise |
| 2.2 | 1-fixed + random | Fixed | Stepwise |
| **3.5-bridge** | Semi-random | Varied (5 templates) | Stepwise |
| **3.8-hybrid** | Semi-random | Varied | Python + LLM |
| **4-llm** | Semi-random | Varied | LLM only |

## Reward Wrappers

### Stepwise (Python)
Dense, rule-based reward computed every step with no LLM calls.

```
reward = coin_reward + Δpotential + step_penalty
```

| Event | Reward |
|---|---|
| Collect required coin | +0.25 |
| Collect extra / wrong coin | −0.25 |
| Δ potential (distance shaping) | α · Δdist |
| Every step | −0.01 |

### Hybrid (Python + LLM)
Same per-step shaping as above, plus an LLM-evaluated episodic score added at the end of each episode.

### LLM Episodic
Zero reward every step; a single LLM call at episode end returns the score.

### LLM Stepwise
One LLM call per step plus one at episode end.  Python pre-computes movement facts (distances, counts); the LLM assigns a reward based on open-ended guidelines.

## Environment Details

| Property | Value |
|---|---|
| Grid size | 5 × 5 |
| Max steps per episode | 30 |
| Coins per episode | 5 |
| Coin colours | red · green · blue |
| Action space | Discrete(4) — up / down / left / right |
| Observation | `obs_grid` (2 × 5 × 5) + `instruction_emb` (384-d) |
| Instruction encoder | `all-MiniLM-L6-v2` (SentenceTransformer) |

The two-channel grid observation encodes:
- **Channel 0** — coin colour index (0 = empty, 1 = red, 2 = green, 3 = blue)
- **Channel 1** — agent position mask

## Setup

### 1 · Clone and install dependencies

```bash
git clone https://github.com/YOUR_USERNAME/reinforcement-learning-llm-reward-coingrid.git
cd reinforcement-learning-llm-reward-coingrid

pip install -r requirements.txt
```

### 2 · Install and start Ollama

Download Ollama from [ollama.com](https://ollama.com) and pull the model used for reward evaluation:

```bash
ollama pull gemma3:12b
ollama serve          # starts the local API on http://localhost:11434
```

### 3 · Run training

**Full curriculum (CLI):**
```bash
python scripts/train.py
```

**Specific stages only:**
```bash
python scripts/train.py --stages 3.5-bridge 3.8-hybrid 4-llm
```

**Override timesteps and save model:**
```bash
python scripts/train.py \
    --stages 3.5-bridge 3.8-hybrid 4-llm \
    --timesteps 3.5-bridge:50000 3.8-hybrid:30000 4-llm:10000 \
    --save-model models/final_ppo \
    --save-plot results/reward_curve.png
```

**Notebook:**

Open `notebooks/coingrid_llm_rl.ipynb` in Jupyter for the original research notebook with all experimental stages.

## Usage (Python API)

```python
from coingrid_llm_rl.env.coingrid_env import CoinGridEnv
from coingrid_llm_rl.wrappers.hybrid import HybridRewardWrapper
from coingrid_llm_rl.training.curriculum import run_full_curriculum
from coingrid_llm_rl.training.plotting import plot_stagewise_rewards

# Run full curriculum
model, stagewise_rewards = run_full_curriculum()

# Plot results — highlight LLM stages
plot_stagewise_rewards(
    stagewise_rewards,
    selected_stages=["3.5-bridge", "3.8-hybrid", "4-llm"],
)

# Use the environment directly
env = HybridRewardWrapper(
    CoinGridEnv(layout_mode="semi-random-1fixed", varied_instructions=True)
)
obs, info = env.reset()
obs, reward, done, truncated, info = env.step(env.action_space.sample())
```

## LLM Prompt Design

The episodic evaluator prompt enforces a deterministic scoring formula:

```
score = 1.0 − 0.5 × total_missed − 0.25 × total_extra
clipped to [−1.0, 1.0]
```

The per-step prompt provides the LLM with movement context (previous/new position, distances to required coins, coins collected this step) and asks it to assign a reward following open-ended guidelines — rewarding progress toward the goal and penalising detours.
